after preprocess: A Schematron Schema for any EAD to validate for EHRI

Version: 0.1

schematron to validate EAD files after preprocess, so they will be imported without any impediments.

Version History:

0.1 (2014-10-07) : Initial rules

  1. EAD header
  2. did's
  3. archdesc's and c-levels

EAD header

the EAD header contains useful information, like language of description, eadid etc.

  1. MUST: eadid MUST contain text
  2. SHOULD: eadheader SHOULD specify a langusage
  3. MUST: language MUST contain a langcode attribute

did's

  1. SHOULD: a did SHOULD have a unitdate
  2. SHOULD: a did SHOULD have a non-empty physdesc-extent
  3. MUST: a did MUST have a unitid
  4. SHOULD: a did SHOULD have a unittitle
  5. MUST: unitid's MUST be unique within one eadfile
  6. MUST: a unitid MUST not be empty
  7. SHOULD: a unittitle SHOULD not be empty

archdesc's and c-levels

  1. MUST: archdesc MUST have a level-attribute
  2. SHOULD: archdesc SHOULD have a non-empty origination
  3. MUST: c01 MUST have a level-attribute
  4. MUST: c02 MUST have a level-attribute
  5. MUST: c03 MUST have a level-attribute
  6. MUST: c04 MUST have a level-attribute
  7. MUST: c05 MUST have a level-attribute
  8. MUST: c06 MUST have a level-attribute
